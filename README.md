# Claude Agent State Applet

Cinnamon panel applet that shows live state of every running Claude Code session as a colored dot — one dot per terminal, grouped by project.

```
┌─────────────────────────────────────────────────────────────────┐
│ Cinnamon panel                                                  │
│  agent-state  ●  |  my-api  ●  ●  |  frontend  ●              │
│  (project A)      (project B)       (project C)                 │
└─────────────────────────────────────────────────────────────────┘
  click label → focus IDE window
  click dot   → focus IDE window (future: focus terminal tab)
```

Dot colors:

| Color  | State                  |
|--------|------------------------|
| grey   | initialized            |
| yellow | working (tool running) |
| blue   | asking user            |
| orange | waiting for approval   |
| green  | done (turn complete)   |

## Architecture

```
┌─────────────────────────────────────────────────────────────────────┐
│  IntelliJ IDEA  (one window per project)                            │
│  ┌──────────────────────────────────────────────────────────────┐   │
│  │  Terminal tab: cc-a1b2c3d4                                   │   │
│  │  $ claude                                                    │   │
│  │     │                                                        │   │
│  │     └─ on every hook event ──► ~/.claude/hooks/state-report.py  │
│  └──────────────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────────────┘
          │  HTTP POST :7855/agent  (JSON)
          ▼
┌──────────────────────────────────────┐
│  claude-state-server.service         │  systemd user service
│  server/claude_state_server.py       │  always running
│  • tracks agents by PID             │
│  • PID liveness checks every 5s     │
│  • POST /agent  — update state      │
│  • POST /focus  — wmctrl focus      │
│  • GET  /status — debug dump        │
└──────────┬───────────────────────────┘
           │  atomic write  (os.replace)
           ▼
     /tmp/claude-agents.json
           │
           │  inotify (+ 3s fallback poll)
           ▼
┌──────────────────────────────────────┐
│  Cinnamon Applet                     │  runs inside Cinnamon shell
│  applet/applet.js                    │
│  • groups agents by project_root    │
│  • colored dots + tooltips          │
│  • desktop notifications            │
└──────────────────────────────────────┘
```

### Component locations (after install)

| Component | Source | Deployed to |
|-----------|--------|-------------|
| Applet JS | `applet/applet.js` | `~/.local/share/cinnamon/applets/claude-agent-state@tommasnik/` |
| Server | `server/claude_state_server.py` | run in-place (path baked into service file) |
| Service | generated by `install.sh` | `~/.config/systemd/user/claude-state-server.service` |
| Hook | `~/.claude/hooks/state-report.py` | already in final location |
| Hook config | `~/.claude/settings.json` | already in final location |

### Data flow per Claude event

```
Claude hook fires (PreToolUse, PostToolUse, Notification, Stop, …)
  → state-report.py runs (subprocess, ~50 ms)
     → detects: claude PID, cwd, project_root, tty, window_id (via wmctrl)
     → POST /agent  →  server updates in-memory dict
                    →  writes /tmp/claude-agents.json atomically
                    →  inotify wakes applet within ~30 ms
                    →  applet redraws dots
```

## Installation

```bash
git clone <repo> ~/code/agent-state-applet
cd ~/code/agent-state-applet
./install.sh
```

`install.sh` does these steps:

1. Copies `applet/applet.js` + `metadata.json` to `~/.local/share/cinnamon/applets/claude-agent-state@tommasnik/`
2. Creates `~/.config/systemd/user/claude-state-server.service` from the template
3. `systemctl --user enable --now claude-state-server`
4. Restarts Cinnamon (`cinnamon --replace`)

After install, **add the applet to the panel manually**:
> Right-click panel → Add applets → search "Claude Agent State" → Add

### Hook setup (one-time, manual)

The hook is already at `~/.claude/hooks/state-report.py`. It must be registered in `~/.claude/settings.json`. If it's missing:

```json
{
  "hooks": {
    "SessionStart":     [{"hooks": [{"type": "command", "command": "python3 ~/.claude/hooks/state-report.py"}]}],
    "UserPromptSubmit": [{"hooks": [{"type": "command", "command": "python3 ~/.claude/hooks/state-report.py"}]}],
    "PreToolUse":       [{"hooks": [{"type": "command", "command": "python3 ~/.claude/hooks/state-report.py"}]}],
    "PostToolUse":      [{"hooks": [{"type": "command", "command": "python3 ~/.claude/hooks/state-report.py"}]}],
    "Notification":     [{"hooks": [{"type": "command", "command": "python3 ~/.claude/hooks/state-report.py"}]}],
    "Stop":             [{"hooks": [{"type": "command", "command": "python3 ~/.claude/hooks/state-report.py"}]}],
    "SubagentStop":     [{"hooks": [{"type": "command", "command": "python3 ~/.claude/hooks/state-report.py"}]}]
  }
}
```

### Dependencies

```bash
sudo apt install wmctrl libnotify-bin
# wmctrl  — window focus + PID-to-window mapping
# notify-send — desktop notifications (libnotify-bin)
```

## Updating components

### Applet changed (`applet/applet.js`)

```bash
cp applet/applet.js ~/.local/share/cinnamon/applets/claude-agent-state@tommasnik/
# (or just run: make applet)
# Then in Cinnamon: remove applet from panel and re-add it.
# OR: right-click panel → Troubleshoot → Restart Cinnamon
```

### Server changed (`server/claude_state_server.py`)

```bash
systemctl --user restart claude-state-server
# No applet restart needed — server writes the same state file.
```

### Hook changed (`~/.claude/hooks/state-report.py`)

No restart needed. The hook is executed fresh as a subprocess on every Claude event.

### Service file changed (template)

```bash
sed "s|__SERVER_PATH__|$(pwd)/server/claude_state_server.py|g" \
    server/claude-state-server.service.template \
    > ~/.config/systemd/user/claude-state-server.service
systemctl --user daemon-reload
systemctl --user restart claude-state-server
```

## Operations

```bash
# Check server status
systemctl --user status claude-state-server

# Live server logs
journalctl --user -u claude-state-server -f

# Current agent state (JSON)
curl -s http://127.0.0.1:7855/status | python3 -m json.tool

# Manually trigger a state update (testing)
curl -s -X POST http://127.0.0.1:7855/agent \
  -H 'Content-Type: application/json' \
  -d '{"pid":9999,"cwd":"/tmp/test","state":"working","hook_event":"PreToolUse","tool_name":"Bash","session_id":"test1234","project_root":"/tmp/test","tty":""}'
```

## Grouping logic

Agents are grouped by **`project_root`** (the nearest ancestor directory that contains `.git`, `.idea`, `pyproject.toml`, `package.json`, `Cargo.toml`, `go.mod`, or `pom.xml`). Fallback: `window_id` (X11 window of IntelliJ), then ungrouped.

The project label in the panel shows the name of the `project_root` directory. Clicking the label focuses the corresponding IntelliJ window (`wmctrl -i -a window_id`).

## Notifications

Sent via `notify-send` when:
- State transitions to `done` — "Claude finished"
- State transitions to `waiting_for_approval` — "Needs approval" (critical urgency)

Notifications are suppressed if the agent's IDE window is already focused.

## `/clear` behavior

When `/clear` is run inside Claude Code, the old session fires a `Stop` hook (`done` state) and a new `SessionStart` fires on the same PTY. The server detects the same TTY and removes the old `done` agent, so the applet shows the new session immediately without a gap.
